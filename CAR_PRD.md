# 차량관리 시스템 - PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 프로젝트명
**caroom** - 차량관리 및 주차장 예약 시스템

### 프로젝트 개요
caroom은 주차장에 차가 있는지 없는지를 확인하고 빈 곳에 예약제로 주차장을 예약할 수 있는 시스템입니다. 실시간 주차장 정보를 확인하고 빠르게 주차 공간을 예약할 수 있습니다.

### 목표
- 주차장 실시간 상태 파악
- 주차 공간 예약 기능 제공
- 차량 정보 중앙 관리
- 주차장 운영 효율성 증대
- 주차 시간 예측 및 계획 기능

---

## 2. 타겟 사용자

- **관리자**: 주차장 상태 모니터링 및 차량 관리
- **보안요원**: 출입 관리 및 주차 현황 감시

## 4. 주차장 구조

### 4.1 주차장 레이아웃
- **1층**: 20개 주차 공간 (1-1 ~ 1-20)
- **2층**: 20개 주차 공간 (2-1 ~ 2-20)
- **총 40개 주차 공간**
- 현재 등록 차량: 20대

### 4.2 주차 공간 상태
- **점유됨**: 현재 차량이 주차 중 (색상: 빨강)
- **예약됨**: 사용자가 미리 예약한 공간 (색상: 오렌지)
- **빈 공간**: 주차 가능한 공간 (색상: 초록)

---

## 5. 핵심 기능

### 5.1 주차장 현황 대시보드
- **실시간 레이아웃 뷰**: 주차장 평면도에서 각 주차 공간의 상태 시각화
- **층별 상태 표시**: 1층, 2층별로 점유율 표시
- **공간별 상세정보**: 각 주차 공간 클릭 시 현재 점유 차량 정보 표시
- **통계 정보**: 점유율, 예약 현황, 빈 공간 개수 표시

### 5.2 주차 공간 예약
- **빈 공간 클릭**: 주차 가능한 공간을 선택하여 예약
- **예약 정보 입력**:
  - 차량 선택 (등록된 차량 목록)
  - 예약 시작 시간
  - 예약 종료 시간 (또는 기간)
  - 특별 요청사항
- **예약 확정**: 한 번의 클릭으로 예약 완료
- **예약 시간 유효성 검증**: 충돌하는 예약 방지

### 5.3 차량 정보 관리
- **차량 목록 조회**: 모든 등록 차량 목록 조회
- **차량 상세정보**: 차량 번호, 모델명, 연식, 종류, 색상, 연비, 상태 등
- **차량 추가**: 새로운 차량 등록
- **차량 수정**: 차량 정보 업데이트
- **차량 삭제**: 사용 중지된 차량 삭제

### 5.4 나의 예약 관리
- **예약 목록 조회**: 사용자가 한 예약 현황
- **예약 수정**: 기존 예약의 시간 및 위치 변경
- **예약 취소**: 기존 예약 취소
- **예약 확인 알림**: 예약 시간 임박 시 알림

### 5.5 주차장 관리 (관리자)
- **차량 위치 등록/수정**: 차량의 현재 주차 위치 설정
- **차량 위치 변경**: 차량 이동 시 위치 업데이트
- **차량 입출 로그**: 모든 주차/출차 기록 조회
- **주차 통계**: 일일/월별 주차 통계 분석

## 7. 데이터베이스 스키마

### 7.1 테이블: `users`
```
- user_id (PK)
- email (string) - 이메일 (유니크)
- name (string) - 사용자 이름
- department (string) - 부서
- phone (string) - 연락처
- is_admin (boolean) - 관리자 여부
- created_at (timestamp)
- updated_at (timestamp)
```

### 7.2 테이블: `vehicles`
```
- vehicle_id (PK)
- license_plate (string) - 차량번호 (유니크)
- model_name (string) - 차량명/모델명
- year (integer) - 연식
- vehicle_type (string) - 종류 (세단, 중형, 소형 등)
- grade (integer) - 등급
- color (string) - 색상
- fuel_type (string) - 연료 (가솔린, 디젤, 전기, 하이브리드)
- mileage_efficiency (string) - 연비 등급
- status (enum) - 상태 (정상, 수리 중, 폐기)
- owner_id (FK) - 소유자 ID
- created_at (timestamp)
- updated_at (timestamp)
```

### 7.3 테이블: `parking_spaces`
```
- space_id (PK)
- floor (integer) - 층 (1, 2)
- space_number (integer) - 공간 번호 (1-20)
- location_code (string) - 위치 코드 (예: 1-1, 2-5)
- is_active (boolean) - 사용 가능 여부
- created_at (timestamp)
- updated_at (timestamp)
```

### 7.4 테이블: `parking_reservations`
```
- reservation_id (PK)
- space_id (FK) - 주차 공간 ID
- vehicle_id (FK) - 차량 ID
- user_id (FK) - 사용자 ID
- start_time (timestamp) - 예약 시작 시간
- end_time (timestamp) - 예약 종료 시간
- special_requests (text) - 특별 요청사항
- status (enum) - 상태 (scheduled, active, completed, cancelled)
- created_at (timestamp)
- updated_at (timestamp)
```

### 7.5 테이블: `parking_logs`
```
- log_id (PK)
- vehicle_id (FK) - 차량 ID
- space_id (FK) - 주차 공간 ID
- check_in_time (timestamp) - 입차 시간
- check_out_time (timestamp) - 출차 시간
- duration_minutes (integer) - 주차 시간(분)
- operator_id (FK) - 담당자 ID
- notes (text) - 특이사항
- created_at (timestamp)
```

## 8. 주요 사용 시나리오

### 시나리오 1: 주차 공간 예약하기
1. 사용자가 대시보드에 접속
2. 주차장 레이아웃 확인
3. 빈 주차 공간(초록색) 클릭
4. 차량 선택 및 예약 시간 입력
5. 예약 확인 및 확정
6. 예약 완료 알림 수신

### 시나리오 2: 실제 주차하기 (관리자)
1. 관리자가 입차 확인
2. 차량 정보 확인
3. 실제 주차 위치 등록
4. 입차 로그 저장
5. 대시보드에 실시간 반영

### 시나리오 3: 주차 차량 확인하기
1. 사용자가 "나의 예약" 메뉴 진입
2. 예정된 주차 현황 확인
3. 현재 차량이 주차된 공간 확인
4. 필요시 예약 수정 또는 취소

### 시나리오 4: 출차하기 (관리자)
1. 관리자가 출차 확인
2. 차량 정보 확인
3. 출차 시간 및 주차 시간 기록
4. 주차비 계산 (향후 기능)
5. 주차 공간 상태 업데이트

## 9. 비기능 요구사항

### 9.1 성능
- 페이지 로딩 시간: 2초 이내
- 예약 처리 시간: 1초 이내
- 실시간 업데이트: 5초 이내

### 9.2 보안
- 사용자 인증 (Supabase Auth)
- 역할 기반 접근 제어 (RBAC)
- HTTPS 통신
- 데이터 암호화

### 9.3 가용성
- 99% 이상의 서비스 가용성
- 자동 백업 (Supabase 관리)

### 9.4 확장성
- 주차 공간 수 증가에 대응 가능
- 차량 수 증가에 대응 가능
- 멀티 건물/지점 확장 가능
